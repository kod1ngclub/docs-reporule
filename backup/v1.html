Repository 구성에 대한 규칙서

Repository 명명 규칙

  • proj 현재 서비스 중인 프로젝트의 Repository
  • toy 프로젝트의 Repository
  • docs 규칙이나 설명서 혹은 실질적으로 작성하는 코드 없이 문서가 주를 이루는 Repository
  • depr Deprecated된 모든 종류의 Repository

Repository의 기술 스택 선정 규칙

  • c/c++ makefile
    low level, CLI, 혹은 고성능 서버
  • markdown markmold (proj-markmold)
    문서
  • markdown toml iced (proj-iced)
    정적 사이트 (혹은 deploy와 rerendering의 필요 주기가 일치하는 경우)
  • scss CSS 프레임워크
  • next.ts vite + typescript
    web-app (혹은 복잡한 web frontend를 필요로하는 경우)
  • clojure (ring)
    상용 프로젝트 (REST 서버의 경우에는 ring 프레임워크 사용)

블로그 작성 규칙

  1. githubname/postname 형태로 branching
  2. 이때, branch이름은 영어 소문자와 /으로만 작성하고, 스페이스는 -으로 대체
  3. common markdown을 준수하여 글 작성
  4. branch를 push
